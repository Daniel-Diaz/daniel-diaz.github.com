Maximum number of tries

-> IO (Either String Bool) 

Ask the AwayServer to delete a full chunk repeatedly until the server reports that it was successfully removed, or the client gets tired of asking (maximum number of tries). Note that it won't try again if an error is received. Only if the request reached the server, and the server answered that the chunk could not be removed. As in clientDeleteChunk, it returns True when the chunk was successfully removed. If False is returned, it means the maximum number of tries was reached.

Chunk to send

-> IO (Maybe String) 

Send a chunk to AwayServer, and insert it at the given PID and initial time. If any error occurs in the process, a String is returned. Note that if the away server detects a valid header, it will accept the bytestring, even if it contains corrupted data. Make sure that the bytestring you are sending is an encoded binary store before calling this function.

When sending to DeadEnd, this function does nothing (and returns Nothing). Sending a chunk to oblivion is not supposed to be an error (or is it?).
